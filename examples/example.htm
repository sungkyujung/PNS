Introduction to the PNScodes

The PNScodes Toolbox for Matlab was created to facilitate the computation and visualization of Principal Nested Spheres, a non-geodesic decomposition method for directional and planar shape data, proposed in the paper ＾Analysis of Principal Nested Spheres￣ by S. Jung, I. L. Dryden, and J. S. Marron (to appear in Biometrika). Bug fixed 2/28/2012.

List of most used m functions

For directional data:

，      PNSmain: computes PNS from data on a sphere

，      PNSgraphics: draw summary plots of PNS fitting

，      PNSe2s: transform Euclidean-type representation to points on a sphere

，      PNSs2e: transform points on a sphere to Euclidean-type representation

 

For 2-d shape data:

，      PNSshape: computes PNS from shape data and draw summary plots of PNS fitting

，      plotshapes: overlay landmarks

，      plotshapesPNS: overlay shape configurations along principal arcs

，      plotshapesPNSani: Animated shape changes along principal arcs

，      procGPA: procedure for generalize procrustes analysis

 

Worked example 1—random directional data

Random directions in S2, that follows von Mises-Fisher distribution of size n = 20, are generated.  Set itype = 0 for PNS with the tests (1 for PNS - strict small spheres fitting, 2 for Principal Nested Great Spheres (PNGS)). Set significance level alpha = 0.05 with R = 100 (# of bootstrap sampling) for the sequential tests discussed in the paper (Supplementary Material).

data = randvonMisesFisherm(4,20,10);

itype = 0;

alpha = 0.05;

R = 100;

[resmat PNS]=PNSmain(data,itype,alpha,R);

 

The following message is generated:

 Message from PNSmain.m; dataset is on 3-sphere.

  .. sequential tests with significance level 0.05

 3-sphere to 2-sphere, by GREAT sphere, p(LRT) = 0.0011502, p(vMF) = 0.77

 2-sphere to 1-sphere, by GREAT sphere, restricted by testing vMF distn

 

The result is plotted by the following expression.

 

PNSgraphics(resmat,PNS)

 

 

Worked example 2—digit 3 shape data

The shapedata.mat contains several shape datasets which are converted from the shapes package of R (http://cran.r-project.org/web/packages/shapes/index.html).  We use the digit 3 data. The dataset contains n = 30 different sets of landmarks, each of which has k = 13 landmarks in d = 2 dimensions. The dataset in Matlab is a k ／ d ／ n array. We first read the data, plot the raw configurations, and plot the Procrustes fitted configurations.

 

load shapedata

 

[k d n] = size(digit3);

 

joinline = 1:k;

figure(1);clf;

subplot(1,2,1)

plotshapes(digit3,joinline,struct('titlestring','Digit 3: raw configurations'));

rotdigit3 = procGPA(digit3);

subplot(1,2,2)

plotshapes(rotdigit3,joinline,struct('titlestring','Digit 3: Procrustes fitted configurations'));

 

We then apply PNS with base = 0 (Procrustes fit to Procrustes mean) and itype = 0 (PNS with the tests).

 

figure(1);clf;

paramstruct = struct('base',0,...

               'itype',0,...

       'icolor',0,'titlestring','digit3 data');

[resPNS,  PNS]=PNSshape(digit3,paramstruct);

The result is now visualized on top of the overlaid landmarks to give a comprehensive mode of major variation. Let PCnum be 1 for the first component, steps be equi-spaced numbers between -2 and 2 representing -2*standard deviation to +2*standard deviation.

figure(2);clf;

PCnum = 1;

steps = -2:0.25:2;

plotshapesPNS(PCnum,steps,resPNS,PNS,joinline)

 

The mode of variation can also be animated by the following code.

 

figure(3);clf;

savestr = 'Digit3ANI';

plotshapesPNSani(PCnum,steps,savestr,resPNS,PNS,joinline)
